Projects.

Compiler

Used the Rust programming language to implement a compiler from scratch. Featuring an advanced type system, I implemented a custom derivative of the Hindley—Milner type system inference algorithm to automatically infer the types of various language constructs such as bindings (also known as variable declarations), parameter types, and return types. This essentially allows the programmer to omit all types to be inferred by the compiler, and yet, still enjoy the benefits of full static typing.

To avoid the common downfalls of subtyping (ex. classes), and taking the overall design of the compiler into consideration, record row polymorphism is employed, since it is a great alternative to subtyping and properly aligns with the other features.

The compiler was written with soundness in mind, thus, it made sense to focus on the functional programming paradigm. This eases up the overall complexity of the project and aligns with the other features as well.

Given that this project nears the 10,000 lines of code, to ensure smooth refactoring, both unit and end-to-end (E2E) tests are employed. I have consistently worked on this project for the past 3 years.
